# EcoByte 🌿

EcoByte is a web application designed to help you understand and reduce your digital carbon footprint. While most carbon calculators focus on physical travel and diet, EcoByte measures the invisible impact of your digital life—such as streaming, cloud gaming, AI prompts, and online storage.

## 🌍 Chosen Vertical
**Environmental Tech / Sustainability & Digital Well-being**

EcoByte focuses on the intersection of personal technology usage and environmental impact. It aims to raise awareness about the carbon emissions generated by daily digital activities and provides actionable steps to minimize them.

## 🧠 Approach and Logic
EcoByte gamifies and simplifies the process of digital sustainability:
1. **Tracking:** Users log their daily digital habits (e.g., hours of HD video streaming, cloud storage size, gaming time).
2. **Calculation:** A specialized calculation engine converts these activities into estimated kilograms of CO₂ equivalents and computes a personalized Digital Carbon Score.
3. **AI Coaching:** Integrates Gemini AI to analyze user data and provide personalized, actionable recommendations for optimizing digital habits and lowering carbon output.
4. **Gamification & Insights:** Users complete daily "green challenges" to earn XP and build streaks. Detailed charts and trajectory models visualize their long-term environmental impact.

## ⚙️ How the Solution Works
1. **Frontend:** Built with Next.js, React, Tailwind CSS, and Motion for a fast, responsive, and engaging user interface.
2. **Backend & Database:** Uses Next.js API routes and Server Actions connected to a MongoDB database (via Mongoose) to securely store user logs, historical data, and scores.
3. **Authentication:** Integrated with NextAuth.js for secure user sessions.
4. **AI Integration:** Uses `@google/genai` to generate custom tips and coaching based on the user's logged activity trends.
5. **Data Visualization & Export:** Employs Recharts to display historical emission trends and jsPDF to allow users to export comprehensive usage reports.

## 📝 Assumptions Made
- **Emission Factors:** The carbon calculations rely on estimated, standardized averages for energy consumption across data centers, transmission networks, and end-user devices, rather than exact real-time network measurements.
- **Manual Data Entry:** The current model assumes users will proactively and manually log their daily digital activities.
- **Grid Carbon Intensity:** A generalized global average for carbon intensity is assumed for digital energy use, rather than localized energy grid data based on the user's exact geolocation.

## 🚀 Getting Started

First, install dependencies and run the development server:

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.
